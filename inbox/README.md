# Инбокс — куда кидать фото и контент

Один раз кладёшь сюда фото + тексты + список «что публиковать», говоришь мне
**«обработай инбокс»** — я запускаю обработку: заливаю фото, привязываю описания, добавляю в
публикацию, генерирую MCP-карточки и выкладываю в фид (Avito забирает по расписанию).

## Структура
```
inbox/
├── photos/            ← фото товаров. Имя файла = АРТИКУЛ (nc_code). jpg/webp/png.
│                        напр. НС-1763718.jpg   (артикул берёшь из Excel / из строки серии)
├── descriptions/      ← тексты описаний (по одному на серию). Имя — любое, напр. tessey.txt
└── inbox.yaml         ← что публикуем (заполняешь по образцу inbox.example.yaml)
```

## Как заполнять `inbox.yaml`
```yaml
items:
  # 1) Товар В НАЛИЧИИ, но без фото в БД (напр. Ballu Tessey) — даём своё фото:
  - series_key: "rusklimat|ballu|tessey dc"   # ключ серии (колонка key в Excel)
    photos: ["НС-1763718"]                     # артикулы, чьи фото лежат в photos/ (без расширения)
    description: "tessey.txt"                  # файл из descriptions/ (можно опустить — тогда автотекст)

  # 2) Товар ПОД ЗАКАЗ (нет в наличии в БД) — добавляем принудительно с ручной ценой:
  - force_nc: "НС-1690797"                     # артикул товара
    price: 18990                               # ручная цена, ₽
    series_name: "ACE-07 FH/N6"                # имя серии (чтобы было отдельным объявлением)
    photos: ["НС-1690797"]                     # если у него нет фото в БД — положи в photos/
    description: "ace07.txt"
```

## Что происходит при «обработай инбокс» (скрипт `scripts/apply_inbox.py`)
1. Каждое фото из `photos/` → конвертит в jpg → заливает на сервер (`/static/manual-photos/{арт}.jpg`)
   → прописывает в `manual_photos`.
2. Каждое описание из `descriptions/` → копирует в `avito-descriptions/` → привязывает в `manifest.json`.
3. Серии из `items` → добавляет в `selected_series` (или `force_include`, если под заказ).
4. Деплой конфига/описаний → `cards_run` (генерация карточек) → пересборка фида.
5. Avito забирает обновлённый фид на ближайшей часовой выгрузке.

## Коротко
- **Просто фото добавить** товару без картинки → положи `photos/{артикул}.jpg` + строку в `inbox.yaml`.
- **Текст описания** → `descriptions/{имя}.txt` + `description: имя.txt` в item.
- **Опубликовать серию** → укажи `series_key` (или `force_nc`+`price` под заказ).
- Артикулы (nc_code) и ключи серий берёшь из Excel-файлов выбора (`avito-series-*.xlsx`).
