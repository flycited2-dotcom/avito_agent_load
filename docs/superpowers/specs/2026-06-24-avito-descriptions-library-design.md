# Библиотека описаний для Avito

**Статус:** согласовано 24 июня 2026 года.

## Цель

Подготовить короткие, живые и разные описания для текущих объявлений кондиционеров на Avito. Текст должен помогать человеку выбрать модель и написать продавцу, а не повторять сухую карточку товара или превращаться в длинный шаблон.

## Исходные данные

Источник фактов — текущий `feed.xml` на VPS (`/opt/avito-bridge/feed_out/feed.xml`), повторно прочитанный 24 июня 2026 года после расширения тестовой партии. В нём 35 активных объявлений-серий. Библиотека отражает именно этот актуальный снимок, а не первоначальную партию из 20 объявлений.

Для каждого текста используются только сведения из фида: название серии, тип кондиционера, доступные типоразмеры, инверторность и подтверждённые характеристики. Цены не попадают в библиотеку: их обновляет фид, а дублировать меняющуюся цену в описании рискованно.

## Формат

В корне репозитория появится каталог `avito-descriptions/`:

```
avito-descriptions/
  README.md
  01-alfacool-apus.txt
  02-expertair-cyclone-dc-inverter.txt
  ...
```

Каждый `.txt` содержит только готовый для вставки в Avito текст без технического заголовка, YAML-полей и служебных комментариев. `README.md` связывает имя файла с серией, ID текущего объявления, типоразмерами и датой снимка фида.

На первом этапе генератор фида не меняется. Библиотека предназначена для ручного просмотра и вставки. Позднее её можно подключить к генератору отдельной задачей, не изменяя сами тексты.

## Структура и тон описаний

Каждое описание занимает примерно 500–900 символов и строится из четырёх коротких смысловых частей:

1. Живой заход через тип помещения или покупательскую задачу.
2. Фактическая польза конкретной серии: инверторность, формат, диапазон типоразмеров или подтверждённая характеристика.
3. Условия продавца, уже используемые в текущих объявлениях: новый товар, гарантия производителя, доставка по Крыму, помощь с монтажом.
4. Ненавязчивый призыв: написать площадь, особенности помещения и город для подбора мощности, наличия и установки.

Тексты различаются первым абзацем, ритмом, аргументом выбора и формулировкой призыва. Запрещены неподтверждённые свойства, ложная срочность, скидки, неподтверждённые сроки и одинаковые большие блоки «Почему мы».

## Проверка результата

Готовая библиотека считается принятой, если:

- есть 35 `.txt`-файлов — по одному на каждую серию текущего фида;
- `README.md` однозначно сопоставляет каждый файл и объявление;
- каждый текст содержит призыв написать для подбора;
- в текстах нет цен и неподтверждённых технических обещаний;
- описания укладываются в 500–900 символов и не повторяют друг друга дословно.
